# Phase 3 Plan 01: Secondary Pages & Consolidation

## Objective

- **What:** Consolidate owner details into `data/profile.json`, build the About page dynamically rendering biography and milestones, the Resume page with embedded PDF viewer, and the Contact page with direct links.
- **Why:** Delivers all supporting sections of the portfolio, ensuring they pull from configurable metadata and stay consistent.
- **Output:** Functional `about.html`, `resume.html`, and `contact.html` pages.

  </read_first>
  <action>
  - Create directory `data` and `assets`.
  - Create `data/profile.json` holding keys:
    - `name`, `tagline`, `university`, `bio` (markdown biography text)
    - `education` (array of objects with school, degree, gpa, date)
    - `experience` (array of objects with role, company, location, date, bullet points)
    - `interests` (array of string tags)
    - `goals` (array of strings)
    - `profileImage` (path string: `assets/profile.jpg`)
    - `resume` (object with path, lastUpdated)
    - `contact` (object with email, linkedin, github)
  - Create a dummy/empty PDF at `assets/resume.pdf` for testing.
  - Create a mock avatar image at `assets/profile.jpg` (or fallback placeholder).
  </action>
  <acceptance_criteria>
  - `data/profile.json` exists and is valid JSON
  - `assets/resume.pdf` exists
  </acceptance_criteria>
  <verify>Check file existence and JSON syntax</verify>
