# Phase 2 Plan 01: Content Engine & Project Templates

## Objective

- **What:** Create the projects index, template, listing page with category filters, dynamic project details viewer page, client-side Markdown rendering with YAML frontmatter, image grids, and image lightbox.
- **Why:** Delivers the content management logic, enabling the portfolio to show real engineering projects dynamically.
- **Output:** Working `projects.html`, `project.html`, projects index, and template.

  - Implement JS engine in `project.html` to:
    - Get `?project=slug` parameter.
    - Fetch `projects/<slug>/project.md`.
    - Extract YAML frontmatter and Markdown body.
    - Parse YAML using `jsyaml.load()`.
    - Parse Markdown body using `marked.parse()`.
    - Render frontmatter header metadata (title, category, date, tags).
    - Render parsed Markdown HTML into body container.
  - Implement dynamic image grouping to bundle consecutive images prefixed with `2-col:` or `3-col:` in their alt text into row-wrapper layouts.
  - Implement lightbox viewer overlay (modal) in `project.html`. When any image in body is clicked, display it fullscreen with zoom/close controls.
  - Add CSS rules for page metadata header, markdown layouts, responsive images, columns, and lightbox overlay.
  </action>
